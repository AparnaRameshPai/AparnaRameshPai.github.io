Overview

The problem & the solution

Acquiring a new customer costs 5× more than retaining an existing one. Telecom companies lose millions annually to churn. This project builds a system that identifies at-risk customers before they leave — giving the business time to intervene.

Rather than a notebook experiment, FinChurn is built as a production-grade ML system — with a REST API, a Streamlit dashboard, MLflow experiment tracking, Docker containerisation, and a clean separation between training and inference. It mirrors how real ML systems are architected in production.

Engineering decisions

Why I built it this way

scikit-learn Pipeline — single serialised artifact
The preprocessor and model are serialised together as a single model.pkl. This ensures training and inference use identical transformations — a common and costly source of bugs in ML systems deployed to production.
Handling class imbalance with scale_pos_weight
The dataset has a 73/27 split (stayed vs churned). Using scale_pos_weight=2.7 tells XGBoost to penalise misclassifying the minority class more heavily. This is why the model achieves 0.84 ROC-AUC despite the imbalance — and why ROC-AUC was chosen over accuracy as the primary metric.
Production-safe encoding
OneHotEncoder(handle_unknown='ignore') ensures the API never crashes when it receives a category it hasn't seen during training — a real risk in live systems receiving user-submitted data.
API-first dashboard architecture
The Streamlit dashboard calls the FastAPI endpoint rather than loading the model file directly. This means the model can be updated, swapped, or scaled independently of the UI — the same pattern used in real ML platforms.

How to run

Getting it running locally

# 1. Clone and install
git clone https://github.com/AparnaRameshPai/MLflow-project.git
cd MLflow-project
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

# 2. Train the model
python -m src.models.train

# 3. Start the API
uvicorn src.api.main:app --reload

# 4. Launch the dashboard
streamlit run dashboard/app.py

# 5. View MLflow experiment tracking
mlflow ui
# → http://127.0.0.1:5000

# 6. Or run everything with Docker
docker build -t finchurn .
docker run -p 8000:8000 finchurn

Dataset

IBM Telco Customer Churn

7,043 telecom customers, 20 features, 26.5% churn rate. Features include contract type, tenure, monthly charges, and service subscriptions. Sourced from Kaggle.
